Filtering

Our capture file can be quite large and quickly become far to much to read but by filtering we can limit what we can see at any one time. A simple filter would be to limit the traffic we are looking at to a specific ip address ip.addr==192.192.192.192. This could be refined to say look only at traffic with a specific source and a specific destination address ip.src == 192.192.192.192 and ip.dst == 192.10.10.10

As wll as filtering by Ip we can filter based on protocol tcp.port eq port number or protocal name or udp.port eq port number or protocol name.

Exploring a packet

Each line in wireshark represents a packet, each one will contain details, open by clicking on the small down arrow? This exposes the the layers of the OSI model.

ARP traffic - Address Resolution Protocol a layer 2 protocol that is used to connect IP addresses with MAC addresses. The message type can be Request (1) or Reply (2).

ICMP traffic - Internet Control Message Protocol used with ping and traceroute. Type 8 is a request and type 0 reply packet.

TCP traffic - Transmission Control Protocol handles the delivery of packets and includes sequencing and errors, TCP sends a lot of traffic so it may be necessary to futher filter the results. Common packets might be the 3 way handshake SYN SYNACK ACK.

DNS traffic - Domain Name Service protocol is used to resolve names with IP addresses, these will be query or response.

HTTP traffic - Hypertext Transfer Protocol is a commonly used port for the www. HTTP is used to send GET and POST requests to a web server, useful data includes Request URI, File Data and Server. Using Statistics>Protocol Hierarchy to display

HTTPS traffic - Hypertext Transfer Protocol Secure - as the name suggests this is secure that means that the data is encrypted however before sending encrypted data the client and server need to agree upon a series of steps: Protocol version, crypo algorithm as well as creating the secure tunnel with a public key, all of this has to be done without the encryption.
