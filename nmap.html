mapping tool and port scanner.

Nmap.

By using IP packets it trys ways of determing what hosts are available on a network. It can futher discover what opperating systems are in use and what services might be available. Available as a command line tool or with an interface (Zenmap). Ncat is a flexible data transfer tool. Ndiff a utility for comparing results. Nping is a packet generation and response analysis tool.

The sections below cover just some of the switches that can be deloped by Nmap.

Host Discovery Switches

Basic Scan - nmap .

List Scan - (-sL) list each host of the specified network and a reverse DNS lookup to learn the hosts names -n will supress the DNS lookup. Nmap will not actually scan the port

Ping Scan or Ping Sweep - (-sn) turns off the port scanning and list those hosts that responded to the host discovery probes.

TCP SYN Ping - (-PS) sends empty TCP packets witht the SYN flag set to test for a three way handshake expecting a SYN/ACK from a live port but even an RST response tells us the port is active.

TCP ACK Ping - (-PA) by sending an ACK most hosts will respond with a RST as no connection is established. This proves that the port is active and listening.

UDP Ping - (-PU) a normally empty packet is sent to test for a response, we would not normally expect an active port to respond but if the port is closed we will get an ICMP unreachable packet. Note firewalls often ignore UDP and allow port detection.

ARP scan - (-PR) only carry out an ARP scan, this only works on the local network or subnet as yourself

ICMP echo request - (-PE) nmap will send an echo request and expects a ICMP echo reply if the machine is online. Many firewalls will block ICMP.

ICMP timestamp requests - (-PP) nmap will use a timestamp request in an attempt to bypass firewall blocking.

ICMP address mask queries - (-PM) nmap will use this request to try and get a response.

others - a range of other switches exist on top of the ones above.

Port States

Open - the port is activly receiving (TCP/UDP/SCTP) .

Closed - the port is accessable but no application is listening..

Filtered - nmap can not determine if the port is open or closed..

Unfiltered - the port is accesible but Nmap is unable to detect if the port is open or closed..

Open/filtered - nmap can not determine if the port is open or filtered.

Closed/filtered - nmap is unsure if the port is closed or filtered.

Port Scanning Switches

all ports - (-p-) scan all ports.

port range - (-p1-1023) scan all ports from 1 to 1023.

100 common ports - (-F) nmap wil scan the 100 most common ports.

scan ports in order - (-r) nmap will scan the ports in consecutive order.

scan speed - (-T0-5) -T0 is the slowest and least likly to be detected while -T5 is very fast but might be unreliable.

Port Scanning Types

TCP SYN scan - (-sS) never completing the handshake this is fast and stealthy..

TCP Connect scan - (-sT) nmap will complete the connection to the open port but this takes longer and the traget system is likly to log the cannection.

UDP scan - (-sU) as open and filtred UDP ports rarely send any response this can be a very slow scan.

TCP NULL, Fin and Xmas scans - (-sN) (-sF) (-sX) nmap sets varrious flags to elicite some response fromt the target and can bypass some firewalls.

TCP ACK scan - (-sA) nmap uses this scan to map firewall rulesets.

Others - nmap has a range of other specialized scans beyond the above list

Service and OS detection switches and more

Version Detection - (-sV) nmap looks for the services and version numbers that are running on a port. We can also add the intensity with (--version-intensity-level) level can be light intensity 2 or all intensity 9. Note this is a non stealthy as full handshake has to be established

Operating System detection - (-O) nmap will attempt to detect the OS and will also guess the version number.

Trace Route - (--traceroute) nmap will try to establish the route taken between you and the taraget IP.

Scripts - (-sC) nmap will use a set of default scripts to establish more information.

Setting Scripts - (--script "script-name") will allow you to set specific scripts to run. You can add to the library or choose from those installed in /usr/share/nmap/scripts.

Saving Output - (-oN filename) will save a normal output. (-oG filename) will save the output in a grepable format. (-oX filename) will cave the output in and XML format.
