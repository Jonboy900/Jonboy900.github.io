OWASP Top Ten.

This identifies and ranks the most critical security risks to web application. At time of writing this refenences the 2021 tables.

1 - Broken Access Control

This covers the password login authenticication system. Brute force attacks with mulipule attempts can be stopped with auto lockouts after a certain number of attempts. Weak credentials and common passwords can be improved by enforcing a strong password policy. Weak session cookies can be mitigated with multi factor authenticication.

2 - Crytographic Failures

Previously known as Sensitive Data Exposure, this covers a range of data that is exposed when a webapp ends up giving it away possibly make the database available or by failing to encrypt or weakly encrypting the data when transmiting it.

3 - Injection

Where user controlled input is interpreted as actual commands or parameters by the application. SQL injection occurs when input is passed to a query, this can allow an attacker to access, modify or delete data in a database. Command Injection is when user input is passed allowing execution of system commands. To defend against this the input data needs to be cleaned or restricted.

4 - Insecure Design

This new category relates to design and architectural flaws. This requires a change in the coding of programs and systems rather than patching it up afterwards. Security needs to be built into the design process and not an add-on to the software.

5 - Security Misconfiguation

This now includes XML External Entity attack (XXE). XML is the markup for cross platform delivery, used for storing and transporting data. It is possible to get xml to perform a read opperation when it is read on the target machine, this provides an opportunity for the target machine to display data that it was not meant to have displayed
