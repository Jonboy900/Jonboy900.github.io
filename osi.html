OSI Model, how computers
communicate.

Open Systems Interconnection Model (OSI).

Every system that can talk or share data uses this model to pass the information to other systems. The model is split into seven distinct layers each part handling different roles in the process of communicating.

Layer 1 - Physical

The Physical layer is the signalling required to transmit and receive the data between machines. The data can be transferred over different mediums: radio waves including bluetooth, wires or fibre optic. Different protocols will handle this actual communication.

Layer 2 - Link

The Link layer is responsible for sending the packets of data in the correct format for the Physical Layer to be able to pass, Ethernet, Wifi are some of the key protocols here.

Layer 3 - Network

The Network layer handles the routing of packages around networks, here the Internet Protocal (IP) addressing protocols reside along with others like ICMP and IPSec.

Layer 4 - Transport

The Transport layer represents applications on the network. Using TCP (Transmission Control Protocol), UDP (User Datagram Protocol) and modern QUIC protocol.

Layer 5 - Session

The Session layer is responsible for establishing, maintaining and terminating connections Here are protocols like SOCKS, NetBIOS and SIP.

Layer 6 - Presentation

The Presentation layer is responsible for adapting, transforming and translating data. Dealing with encoding ASCII and UTF, encryption SSL and TLS as well as compression.

Layer 7 - Application

Most Internet browsers would be found on this layer with the Protocols of HTTP, FTP, SNMP all working here.
