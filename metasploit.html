Metasploit.

A collection of tested exploits with an extended set of tools used to identify vulnerablilities on servers and networks.

Suits of tools - modules

Exploit - for taking advantage of weekness in a target system.

Payload - a set of codes which can be run after exploting and weekness in a system.

Encoder - for converting information.

Listeners - .

NOP - .

Auxillary - An extended set of further tools.

Post - for post exploitation including gaining futher information on a target system.

Set up required

initialize the data base - use the msfdb init command.

start Metasploit - use the msfconsole command.

Simple Example - Bolt on THM

msfconsole - to start using metasploit.

db_nmap -sV (target IP) - to get a map of the ports and services running. Port 8000 is open and looks like a web server.

Browse the web site - navigate to (target IP):8000 and have a look around we can get the admin name and password in blogs here as well as seeing that the CMS is Bolt.

Try and login - need to find the login page. Google bolt cms to find default login page. Browse to (target IP):8000/bolt and login with admin name and password note can see version of Bolt running here.

search bolt in metasploit - to look for any known exploits, this returns 2 known exploits including the exploit/path name and which versions of Bolt they work on.

info (exploit/path) - will give us a lot of information on how to use this and provide the EDB-ID of the exploit is we want to look for more info on this exploit.

use (exploit/path) - this will set us to using the exploit part of metasploit. Now we will need to set some variables LHOST (that is our attack box) RHOST (target machine) USERNAME and PASSWORD.

set (VAR value) - to set up the variables.

run - will start the exploit with the variables we have set and if successful will create a shell for us.

Bash commands to look around - ls, pwd, help, cd/home, ls and cat (filename) will all be useful here to find the flag.
