Metasploit.

A collection of tested exploits with an extended set of tools used to identify vulnerabilities on servers and networks. The tools are divided into some separate sections and you can background these sessions with CTRL Z to allow you to jump into other parts. Sessions will list the open sessions for you and sessions -i # will take you back to the specified session.

Suits of tools - modules

Exploit - for taking advantage of weakness in a target system.

Payload - a set of codes which can be run after exploiting and weakness in a system.

Encoder - this will allow us to set the exploits.

Listeners - .

NOP - this is no operation tasks.

Auxiliary - An extended set of further tools.

Post - for post exploitation including gaining further information on a target system.

Commands

start Metasploit - use the msfconsole command.

initialize the data base - use the msfdb init command and db_status to check the database connection.

help or ? - to bring up the menu of available commands.

connect - to make a quick connection with a host to verify that we can talk with it.

db_nmap - to run an Nmap (add the flags you want) to map ports and services.

search - to look for an exploit, usage search "exploit_name" name must be in double quotes.

use - to use an exploit (not actually to run it) but so that we can set the variables we might need to run it and to find out more about it. Usage use #.

info - about the current module selected or a specific one, this will give a report on what variables need to be set to use an exploit.

set - to set the variables in the exploit. Usage VARIABLENAME value

get - will show a variable named.

run - to run an exploit with the variables set.

Simple Example - Bolt on THM

msfconsole - to start using metasploit.

db_nmap -sV (target IP) - to get a map of the ports and services running. Port 8000 is open and looks like a web server.

Browse the web site - navigate to (target IP):8000 and have a look around we can get the admin name and password in blogs here as well as seeing that the CMS is Bolt.

Try and login - need to find the login page. Google bolt cms to find default login page. Browse to (target IP):8000/bolt and login with admin name and password note can see version of Bolt running here.

search bolt in metasploit - to look for any known exploits, this returns 2 known exploits including the exploit/path name and which versions of Bolt they work on.

info (exploit/path) - will give us a lot of information on how to use this and provide the EDB-ID of the exploit is we want to look for more info on this exploit.

use (exploit/path) - this will set us to using the exploit part of metasploit. Now we will need to set some variables LHOST (that is our attack box) RHOST (target machine) USERNAME and PASSWORD.

set (VAR value) - to set up the variables.

run - will start the exploit with the variables we have set and if successful will create a shell for us.

Bash commands to look around - ls, pwd, help, cd/home, ls and cat (filename) will all be useful here to find the flag.
